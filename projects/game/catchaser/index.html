Presentation of the project

I wanted to make a little game on unity in one/two days to get used to all the differents tools of unity. Catchaser is really simple, you have a maki in the center of the screen and some cats arrive to try to eat it. The objective of the game is to stop all the cats to eat it by cliking on them. When you click on a cat, the cat run away and you gain 10 points. If a cat reach the maki, you loose the game.

I added a scoreboard to this game so when you loose a game, you can enter your score and it's send into a database. On the main menu of the game you can see the scoreboard which show the top 10 score.

This is a little project that I made really quickly so it's really ugly (especially the music), please ignore that.

Implementation

Like I said in the presentation, I made this project in Unity. I made everything in only one scene which is not the best because I have to activate/desactivate button/object between scene. I know there is better way to do it now but I was learning how to use unity at the time. The difficulty function have a form like : 1 - 1/(1+e^x), with this function, the difficulty grow slowly and stabilize at some point.

I made an API to enter and read score in database. I used Java with the framework Sprint Boot to create it, it's was really easy because spring boot is really useful for a quick creation of api/database. I didn't wanted to have a seperate database with the api so said to spring to create an embedded h2 database to store the data (it's not a memory database but a file so it keeps data when restarting). I really like the api like it is because it's really simple to deploy on a server, I only have to run the jar file and everything is ready to go. The only problem is that I don't secure the api so everyone can add any score on the database which is bad but I don't know yet how to secure this.

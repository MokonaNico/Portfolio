Presentation of the project

The goal of this project was to implement go-Back-N protocol with a congestion control of type Reno. These are protocole types that transmit data in a reliable way on an unreliable channel. To simulate the network, we worked on a simulator made by one of our teacher.

I'm not going to explain in details what is go-Back-N and the congestion control. In short, it's just a way to be sure that the data is well transmited with pipelining (that means that multiple chunk of data can be sent one after the other). To choose the number of data that we want to send, we limit ourselves with a window size. This window's size can be changed by the congestion control to avoid the congestion colapse.

It was a project by group of 2 students. I can't share the source code because it's a university project.

Implementation

The Network simulator was in Java so we implemented the protocol in Java inside the simulator. We also created a bit of code in python to read the data output by the simulator and plot it, we used Mathplotlib for that. It was a bit hard to merge go-Back-N and congestion control but with the help of our notes, we found a way that works pretty well.

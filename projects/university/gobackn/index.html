Presentation of the project

The goal of this project was to implement a protocole of type go-Back-N with a controle of congestion of type Reno. These are type of protocole to transmit data in a reliable way on an unreliable channel. To simulate the network, we worked on a simulator made by one of our teacher.

I'm not going to explain in details what is go-Back-N and the controle of congestion, to be simple, it's just a way to be sure that the data is well transmited with pipelining (that means that multiple chunk of data can be send one after the other). To choose the number of data that we want to send, we limit our self with a window size. This window size can change of size by the controle of congestion to avoid the congestion colapse.

It was a project by group of 2. I can't share the source code because it's a university project.

Implementation

The simulator of Network was in Java so we implement the protocol in Java inside the simulator. We also created a bit of code in python to read the data output by the simulator and plot it, we used Mathplotlib for that. It was a bit hard to merge go-Back-N and controle of congestion but with the help of our notes, we find a way to do it which work pretty much good.

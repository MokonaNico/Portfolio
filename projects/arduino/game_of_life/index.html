Presentation of the project

This project is an implementation of the game of life from Conway. I already did the game of life before with java and javafx but this time I did it on arduino with a led 8x8 matrix, which I find really cool because it's really cute to see all the iterations on a little screen. So I'll brievely explain what is the game of life, it's pretty much simple. So the universe here is a 2d grid and each cells of this grid can have two possible states, live or dead. At each step the state of all the grid is updates following some rules that are :

  1. Any live cell with fewer than two live neighbours dies, as if by underpopulation.
  2. Any live cell with two or three live neighbours lives on to the next generation.
  3. Any live cell with more than three live neighbours dies, as if by overpopulation.
  4. Any dead cell with exactly three live neighbours becomes a live cell, as if by reproduction.

With these simple rules, really impressive patterns can appear, some don't move, other repeat and a few move in a direction. You can see some of them in the image section that I got on my 8x8 led matrix. But it exists a lot more that you can find on the internet if you are interested.

Implementation

So to represent the game of life, I created a class named after it that handle the grid and can generate the next step. Another think that I did was to randomly generated the first step so some really unespected pattern can appears. In the theory, the grid for the game of life have to be infinite, which is impossible with an implementation on a computer but I was really more limited with my 8x8 led matrix. So I decided that the border have to be connected.

For this project I only used a LED 8x8 matrix connected to my arduino, you can get the source code here : Github page
